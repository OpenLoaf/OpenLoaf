## Positioning

一句话定义：

`黑场产品剧场 + 工程器材手册 + 浅灰商店目录收尾`

风格公式：

`纯黑舞台 + 单一硬件主角 + 居中轻字重标题 + 德英混排微文案 + 图解式技术段 + 浅灰大圆角商品卡 + 黑底规格尾段`

和通用 teenage engineering 风格相比，`OP–XY` 更黑、更克制、更像“产品发布展厅”，也更强调“中段讲功能，尾段讲购买和参数”。

## Distinctive Traits

### 1. 黑场不是背景，而是舞台

- 首屏和大多数叙事段都落在 `#000000` 上。
- 产品像舞台中央的唯一展品，背景几乎不承担情绪，只承担衬托。
- 视觉冲击主要来自对象的材质、反射、灰度层级，不来自大色块或特效。

### 2. 章节标题像展签，不像营销大字报

- 标题大量使用 `light / thin` 的无衬线。
- 大标题经常居中、留很大空气、字重非常轻。
- 章节名偏短，甚至故意保留一点俏皮或异语感，例如德语短句。
- 正文仍然技术化，避免大段品牌宣言。

### 3. 页面节奏是“黑 -> 黑 -> 黑 -> 灰 -> 黑”

- 前中段持续在黑底上推进产品、功能和接口。
- 接近购买区域时，突然切到浅灰大卡，制造强节奏反差。
- 最后再回到黑底技术规格，形成冷静收束。

### 4. 重点不是 UI 组件，而是对象编排

- 大部分信息块并不依赖常规 SaaS 卡片。
- 更常见的是：
  `居中文字块 / 器材特写 / 接口标注 / 技术词条列表 / 目录式商品卡`

## Layout System

页面源码暴露出的骨架非常明确：

- `12` 列布局
- 桌面内容宽度基准约 `980px`
- 左右边距基准约 `45px`
- gutter 基准约 `10px`
- 垂直节奏大量以 `10 / 20 / 40` 的倍数推进

可复用成现代页面规则：

- 桌面主容器：`max-width: 980px` 到 `1120px`
- 页面左右安全边距：`40px - 56px`
- 栅格 gutter：`10px - 16px`
- 大章节垂直间距：`96px - 192px`
- 文本段宽：控制在 `4 - 6` 列，不要铺满
- 购买卡片：允许大面积方卡或近方卡，保留很大的图像占比

### Section Rhythm

优先按这个顺序组织产品页：

1. `黑场首屏`：单一产品主体，文案极少。
2. `功能叙事段`：居中标题 + 一小段技术性说明。
3. `功能清单段`：用短语矩阵列出能力，不做重卡片包装。
4. `图解段`：用线、标签、接口名、步骤名解释设备。
5. `购买段`：切换到浅灰卡片，出现价格和目录式 CTA。
6. `规格段`：黑底双列或多列技术规格列表。

## Color Rules

从页面 CSS 可直接提炼出这组颜色：

- `#000000`：页面主舞台黑
- `#0f0e12`：结构黑、深色按钮、视频控件主色
- `#f5f5f5`：高亮文字、浅白前景
- `#e5e5e5`：商店卡片底、大面积浅灰面
- `#b2b2b2`：正文浅灰、辅助说明
- `#7f7f7f`：弱化说明、技术次级信息
- `#272727`：视频控件暗部
- `#0071bb`：链接或功能性少量点缀
- `#f05a24` / `#b81d13`：只允许语义性极少量使用

使用原则：

- 绝大多数面积保持黑、灰、白。
- 页面氛围由灰度层级构成，不由彩色构成。
- 强调色不要铺面，只能点在链接、状态、产品自身细节或极小功能点上。

## Typography Rules

源码显示其基础字体系偏 `Univers` 风格的超轻无衬线，字号体系非常克制。

复用时遵循：

- 标题：优先 `thin / light`，不要粗黑标题。
- 正文：继续沿用同家族，保持统一。
- 小标题：可以全小写，也可以用很轻的 uppercase。
- 标签：允许 `uppercase + 稍高字距`，但不要做成潮牌海报字。
- 技术词条：用短句、型号、接口名、步骤词。

可替代字体方向：

- Univers
- Neue Haas Grotesk
- Suisse Int'l
- Helvetica Neue
- Nimbus Sans

避免：

- `Inter` 默认态直接上屏
- 过度几何化、互联网感强的标题字体
- 标题和正文像两个不同品牌系统

## Image Direction

`OP–XY` 页面对图像的处理规则很鲜明：

- 产品要像黑场里的展品，而不是货架商品。
- 主图优先近正视角或严谨控制的局部特写。
- 可以使用镜面反射、灰度条、极弱舞台感结构，但不要炫技。
- 产品图大面积占屏，文字退后。
- 不依赖 lifestyle 人像或场景图撑气氛。

具体可复用成：

- 背景：纯黑或极深黑
- 主图：单对象、中心构图或轻偏置构图
- 光线：边缘光、反射光、控制过的金属/塑料表面高光
- 辅助图：按钮、接口、旋钮、屏幕、材质特写

## Component Syntax

### Hero

- 一个主对象就够，不要并列多个卖点模块。
- 标题和副标题都偏轻，甚至比常规品牌页更轻。
- CTA 要弱，常见做法是下划线链接或很小的目录动作。

### Video / Media Block

- 视频控件皮肤也保持工业黑灰。
- 交互存在，但只用来支撑观看，不制造视觉表演。
- 进度条、音量条、播放按钮都属于“器材 UI”，不是娱乐 UI。

### Feature Matrix

- 用居中的短语矩阵列能力点。
- 每一条像规格名，不像广告句。
- 不必套统一卡壳，直接靠间距、对齐和字号建立秩序。

### Diagram Section

- 用标签和对象关系解释接口、步骤、音频路径或交互点。
- 标签短、直、技术化。
- 线条和标注要细，不要用夸张箭头和说明气泡。

### Shop Cards

这是这页最值得单独保存的一条规则：

- 背景切到 `#e5e5e5` 的浅灰大卡。
- 卡片通常接近正方形。
- 圆角明显偏大，但不软萌。
- 图像占比高，信息区很薄。
- 左侧是产品名和价格，右侧是深色胶囊式目录 CTA。

落地建议：

- 卡片圆角：`32px - 56px`
- 卡内留白：`24px - 40px`
- CTA：深黑底、浅灰字、极短动作词，例如 `visit store`

### Technical Specs

- 用黑底双列或多列清单收尾。
- 分类名比参数值更亮。
- 参数值保持灰色，连续排布，像手册，不像营销模块。
- 不要把规格区做成折叠 FAQ。

## Motion Rules

源码 token 显示：

- 页面级过渡约 `0.15s`
- slider 约 `0.35s`
- hover 约 `0.15s`

复用时遵循：

- 只用 `opacity / transform / color`
- hover 只做轻微亮度、对比或位移
- 不用夸张 spring
- 不用重 blur、发光、悬浮感
- 滚动动画可有，但必须少

## Copy Tone

这页的文案语气是少见的 `德语短标题 + 英文技术说明` 混合：

- 标题像展览命名或设备绰号
- 正文像产品说明，但保留一点调侃
- 功能清单像器材 spec
- CTA 像目录动作，不像情绪召唤

可复用原则：

- 用短句
- 少形容词
- 多用对象词、动作词、接口词、功能词
- 让句子像“在描述设备能力”，而不是“在推动转化”

## Reuse Recipe

当用户说“做成 OP–XY 那种页面”时，先套这套最小骨架：

1. `黑底首屏 + 单一产品大图`
2. `居中轻标题 + 极短技术说明`
3. `无卡片能力矩阵`
4. `接口或工作流图解段`
5. `浅灰商店卡片区`
6. `黑底规格清单区`

如果只借气质，不整页复用，则保留这四件事：

- 黑场单对象
- 超轻无衬线排版
- 灰度主导而非彩度主导
- 购买区用浅灰目录卡而非营销按钮

## Don't

- 不要把整页做成普通黑色电商详情页
- 不要把强调色铺满整屏
- 不要使用过多卡片把内容切碎
- 不要在黑底上叠玻璃拟态和蓝紫发光
- 不要用大段品牌宣言挤掉产品主体
- 不要直接复刻对方专有字体、文案和图像
